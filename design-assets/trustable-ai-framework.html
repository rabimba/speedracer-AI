A Domain Expertise Layer
on top of the Google stack — not just a Gemini wrapper.

Today's best telemetry coaches (Garmin Catalyst) tell you what went wrong with numbers, after the fact. This system delivers context-aware coaching as it happens, adapted to driver skill, by routing every decision through a layered domain doctrine — Ross Bentley pedagogy, T-Rod's Sonoma session notes, and track-specific corner doctrine — before any LLM ever speaks. Gemini becomes a tool inside a trust framework, not the source of trust.

What makes this trustable

Three pillars. The Domain Expertise Layer is the load-bearing one — without it, this is just another LLM wrapper.

1. Domain Expertise Layer DEL

Ross Bentley's 7 mental models + T-Rod's 5 Sonoma coaching patterns + corner-by-corner Sonoma doctrine. It is not prompt text — it actively suppresses premature advice, biases hot-path wording, and shapes feedforward per corner and per skill level. Doctrine, not decoration.

2. Split-Brain Coaching Engine

Hot path (12 deterministic rules, <50ms, no I/O). Cold path (gemini-2.5-flash-lite, 2–5s, multi-frame, skill-adapted prompt). Edge reasoner (Gemma 4 E2B-it on-device). Feedforward (geofence, 150m before corner). The hot path NEVER waits for the cold path. P0 safety preempts everything, including the LLM's turn to speak.

3. Trust Gates Before Speech

Timing state machine (OPEN → DELIVERING → COOLDOWN → BLACKOUT during apex). Driver model classifies skill from input smoothness. Bench-safe motion gating suppresses stationary false positives. Confidence thresholds gate every utterance. Silence is a feature.

Domain Expertise Layer — what's actually in it

Implemented in src/data/trackExpertise.ts, src/data/trodCoachingData.ts, and src/utils/coachingKnowledge.ts. Wired into the runtime selector, not just the prompt.

Ross Bentley pedagogy (7 mental models)

  • Friction Circle — clock metaphor for combined load
  • Weight Transfer — seesaw metaphor for chassis balance
  • Trail Braking — handoff from braking to cornering force
  • Vision Drives the Car — eyes lead inputs
  • Maintenance Throttle — small throttle settles the platform
  • Slow In, Fast Out — exit speed beats hero entry
  • One Thing at a Time — cognitive load discipline

T-Rod Sonoma coaching (5 patterns)

  • Throttle Commitment — 100% at apex, not a moment earlier
  • Brake Trace Quality — ski slope, not cliff
  • Delay Early Throttle — wait for apex, then commit hard
  • Distance Is King — shorter path beats higher speed in sweepers
  • Session Learning Sequence — lines → shifts → trail braking → throttle

Sonoma track doctrine

  • Corner dossiers for T2, T3, T3A, T6, T7, T9–10, T11, T12
  • Three timing sectors with sector-specific guidance
  • Sonoma is the primary deployed track in both web registry and native Android catalog
  • Doctrine biases feedforward wording per corner, not generically

Skill-adapted humanization

  • Beginner: "Hold a little brake as you turn in." (feel-based)
  • Intermediate: persona-flavored (AJ, Rachel, Tony, Garmin)
  • Advanced: "Trail off. G-Lat: 0.85. Release linearly to apex." (data)
  • Phase 1 (0–60s) suppresses advanced techniques entirely

Three-tier on-device reasoner cascade

EdgeRuntimeManager.warmupPreferredBackend() probes each backend in order. The first one that responds with a non-UNAVAILABLE / non-ERROR state becomes the active reasoner; the choice is persisted to SharedPreferences so subsequent sessions don't re-probe. Confidence scores per tier: AICore 0.78, LiteRT-LM 0.72, Deterministic baseline.

Tier 1 · AICore
Gemini Nano 4 / Gemma 4 preview Prompt API. Probed first via reflection on com.google.ai.edge.aicore.GenerativeModel. Highest confidence (0.78). Supports HOT, FEEDFORWARD, EDGE paths.
Tier 2 · LiteRT-LM
Gemma 4 E2B-it (.litertlm, sha256-verified). Loaded via MediaPipe tasks.genai.llminference.LlmInference. Pushed to /data/local/tmp/koru/models/gemma-4-e2b-it/ by npm run pixel:e2e:prepare.
Tier 3 · Deterministic
Phrase catalog + 12-rule decision matrix. No LLM. Always available. Guarantees the system never goes silent due to an ML failure — this is the "trustable" floor.

Timing state machine

Implemented

OPEN → DELIVERING → COOLDOWN → BLACKOUT. Beginners get 3s cooldown and full mid-corner blackout. Advanced get 1s cooldown, no blackout. P0 safety bypasses every state.

Priority queue

Implemented

P0 preempts all. Max 5 items, 3s stale expiry. preempt() clears non-safety messages on safety alerts so a "BRAKE!" never sits behind two technique tips.

Driver model + hysteresis

Implemented

10s rolling window (rate-agnostic 8–25Hz). 5s hysteresis prevents skill-class flapping. Smoothness < 0.4 OR coasting > 30% ⇒ BEGINNER.

Bench-safe motion gating

Implemented

Phone-only telemetry coaching is suppressed until real movement is detected. Eliminates the stationary-on-a-desk false-positive coaching class entirely.

Session phase filter

Implemented

Phase 1 (0–60s) suppresses TRAIL_BRAKE, COMMIT, ROTATE, EARLY_THROTTLE, COGNITIVE_OVERLOAD. Advanced drivers skip directly to Phase 3.

Confidence + speed gates

Implemented

Realtime speech hardening interrupts lower-priority speech with higher-priority cues, and gates utterances on confidence. Faster timing for advanced drivers.

Pre-rendered safety MP3s

Planned

AudioContext pre-caching is wired; need to record per-persona BRAKE / OVERSTEER_RECOVERY / COMMIT clips to bypass TTS latency on time-critical calls.

Cold path offline fallback

Future

Pre-computed coaching lookup table per known track for offline replacement of Gemini cold path. Pixel 10 + Gemma 4 evaluated as upgrade over Nano.

Standardization opportunities for cohort cross-pollination

Patterns that should be lifted to cohort level. The DEL pattern transfers across domains; the racing specifics don't.

Common contract: VehicleDataStream

Proposed

Abstract OBD/CAN/BLE behind one interface so the coaching engine never changes when upgrading from OBD to direct CAN bus. Ready for Team 2 (CAN-to-USB) and Team 3 (MoTec).

Shared post-session schema

Open

Per-corner metrics, mistake zones, coaching decisions. Codify the saved-session artifact so AGY pipeline ingestion is identical across all three teams.

Priority tier convention

Reusable

P0 safety / P1 tactical / P2 strategic / P3 encouragement. Trivially portable across domains. Recommend the cohort adopt it for any real-time advisory work.

Time-based windows over frame counts

Reusable

Sensor rates differ (8 Hz OBD vs 25 Hz RaceBox). Always store {time, value} pairs; trim by elapsed time. One decision eliminated an entire bug class.

Domain Expertise Layer pattern

Reusable

Doctrine that filters and shapes LLM output is the trust mechanism. Any cohort domain (medical, fraud, ops) needs this layer between raw model and user.

Mocked telemetry endpoint

In progress

Throttled synthetic stream so all teams can validate ingestion pre-field-test. Owned by Team 1. Standardizing on this unblocks pipeline development for everyone.
